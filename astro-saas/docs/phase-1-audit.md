# Phase 1 Audit and Execution Plan

## Current Repository Reality

The existing root project is a FastAPI backend plus static frontend implementation. It is not an Astro SaaS baseline and has no production-grade auth, billing, or Supabase-integrated data model.

## Salvage Matrix

### Keep

- Vedic domain logic and formula references from `backend/app/ephemeris` and `backend/app/symbolic`.
- Existing request shape and reading structure as temporary compatibility inputs.

### Refactor

- Existing prediction orchestration into API-agnostic services, then adapt into Astro server routes.
- Existing location and chart computation pipeline into typed modules.

### Replace

- Frontend stack: replace static JS UI with Astro + React islands.
- Data layer: replace ad-hoc local SQL bootstrap with Supabase migration and RLS-first schema.
- API surface: replace FastAPI endpoints with Astro API routes.

### Defer (Not deleted)

- Legacy backend and frontend remain in the root for reference during migration.

## Target Architecture

- `src/pages`: web routes and API routes.
- `src/lib/schemas`: typed validators for API payload contracts.
- `src/lib/server`: server-only clients and service orchestration.
- `supabase/migrations`: SQL migrations with auth-safe policies.
- `docs`: implementation notes, rollout decisions, and risk logs.
